Portfolio changes

Problems identified

1. Logo not available
2. Weak page or limited titles.
3. Not strong enough Meta descriptions
4. Long blog pages no real navigation
5. Poor colour contrast used on the form
6. Repetitive alt text
7. Change colour scheme
8. Grammar errors

Problem resolved (Solutions)

1. Used the Favicon as a Logo for the users to acknowledge they are on the right portfolio site, since the logo is now clickable the user can easily navigate back to the home page.
2. The pages appeared to be weak as ‘CAPITAL letters’ were used, so the capital letters are now replaced with lowercase.
3. Meta tags needed to be described to define the web pages for that reason Meta Tag Generator is used to generate Good meta description tags, to help improve the click through rates in the search engines.
4. A long blog posts page is now replaced with one blog page that has 6 blog post clickable card sections, by clicking on the card it will take you to your selected blog’s page.



5. Replaced the background colour for the input text area of the contact form.
6. Improved few obvious alt texts that appeared to be similar.
7. The #CC0055 didn’t go with the colour scheme as it was hard to read over the back background, so it is now replaced with #FFFF98.
8. The Blog posts were reread for grammar check and improved to the best of knowledge.
